## 话题

v1.8→v1.9 hard reset 时，VEM（v1.7）和 Output Contract（v1.6）因超过 150 行限制被整体删除。W-003 要求在 <150 行约束内恢复这两个章节的核心价值。

## Step 1.5 三问自答（自主模式）

1. **这个改动让 skill 更容易用，还是更复杂？** — VEM 把"谁验证、何时验证"从模糊原则变为可操作规则；Output Contract 让输出从随机格式变为可解析。复杂度不增加，准确度提升。→ 更准确 ✓
2. **一个真实 Claude 会 follow 这条新规则吗？** — "先定验证级别再选 Mode"是明确行为指令；`[已验证]/[假设]/[未验证]` 是简单标注约定。→ 是 ✓
3. **如果 Codex 输出不完全符合新规范，用户还能提取有用信息吗？** — VEM 是用户自己的决策框架；Output Contract 即使部分遵守也有价值。→ 是 ✓
